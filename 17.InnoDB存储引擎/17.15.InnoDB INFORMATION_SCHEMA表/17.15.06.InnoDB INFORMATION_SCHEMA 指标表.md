#### 启用、禁用和重置计数器

您可以使用以下变量启用、禁用和重置计数器：

- `innodb_monitor_enable`：启用计数器。

```sql
SET GLOBAL innodb_monitor_enable = [counter-name|module_name|pattern|all];
```
- `innodb_monitor_disable`：禁用计数器。

```sql
SET GLOBAL innodb_monitor_disable = [counter-name|module_name|pattern|all];
```
- `innodb_monitor_reset`：将计数器值重置为零。

```sql
SET GLOBAL innodb_monitor_reset = [counter-name|module_name|pattern|all];
```
- `innodb_monitor_reset_all`：重置所有计数器的值。使用 `innodb_monitor_reset_all` 前必须禁用计数器。

```sql
SET GLOBAL innodb_monitor_reset_all = [counter-name|module_name|pattern|all];
```

计数器和计数器模块也可以在启动时通过 MySQL 服务器配置文件启用。例如，要启用日志模块、`metadata_table_handles_opened` 和 `metadata_table_handles_closed` 计数器，请在 MySQL 服务器配置文件的 `[mysqld]` 部分输入以下行。

```ini
[mysqld]
innodb_monitor_enable = log,metadata_table_handles_opened,metadata_table_handles_closed
```

在配置文件中启用多个计数器或模块时，指定 `innodb_monitor_enable` 变量，后跟以逗号分隔的计数器和模块名称，如上所示。配置文件中只能使用 `innodb_monitor_enable` 变量。`innodb_monitor_disable` 和 `innodb_monitor_reset` 变量仅在命令行上支持。

> **注意**
>
> 因为每个计数器都会增加一定程度的运行时开销，在生产服务器上谨慎使用计数器，以诊断特定问题或监控特定功能。推荐在测试或开发服务器上更广泛地使用计数器。
>

#### 计数器

可用计数器的列表可能会更改。查询 `INFORMATION_SCHEMA INNODB_METRICS` 表，以了解您的 MySQL 服务器版本中可用的计数器。

默认启用的计数器对应于 `SHOW ENGINE INNODB STATUS` 输出中显示的那些。`SHOW ENGINE INNODB STATUS` 输出中显示的计数器总是在系统级别启用，但可以为 `INNODB_METRICS` 表禁用。计数器状态不是持久的。除非另有配置，否则服务器重启时计数器会恢复到其默认的启用或禁用状态。

如果您运行的程序会受到计数器添加或移除的影响，建议您在升级过程中查看发布说明并查询 `INNODB_METRICS` 表，以识别这些变化。

#### 计数器模块

每个计数器都与特定的模块关联。可以使用模块名来启用、禁用或重置特定子系统的所有计数器。例如，使用 `module_dml` 启用与 dml 子系统关联的所有计数器。

```sql
mysql> SET GLOBAL innodb_monitor_enable = module_dml;

mysql> SELECT name, subsystem, status FROM INFORMATION_SCHEMA.INNODB_METRICS
       WHERE subsystem ='dml';
+-------------+-----------+---------+
| name        | subsystem | status  |
+-------------+-----------+---------+
| dml_reads   | dml       | enabled |
| dml_inserts | dml       | enabled |
| dml_deletes | dml       | enabled |
| dml_updates | dml       | enabled |
+-------------+-----------+---------+
```

- `module_compress` (subsystem = compression)
- `module_ddl` (subsystem = ddl)
- `module_dml` (subsystem = dml)

#### 示例 17.11 使用 `INNODB_METRICS` 表计数器

此示例演示了启用、禁用和重置计数器，以及在 `INNODB_METRICS` 表中查询计数器数据。

1. 创建一个简单的 InnoDB 表：

    ```sql
    mysql> USE test;
    Database changed
    
    mysql> CREATE TABLE t1 (c1 INT) ENGINE=INNODB;
    Query OK, 0 rows affected (0.02 sec)
    ```

2. 启用 `dml_inserts` 计数器。

    ```sql
    mysql> SET GLOBAL innodb_monitor_enable = dml_inserts;
    Query OK, 0 rows affected (0.01 sec)
    ```
    可以在 `INNODB_METRICS` 表的 `COMMENT` 列中找到 `dml_inserts` 计数器的描述：

    ```sql
    mysql> SELECT NAME, COMMENT FROM INFORMATION_SCHEMA.INNODB_METRICS WHERE NAME="dml_inserts";
    +-------------+-------------------------+
    | NAME        | COMMENT                 |
    +-------------+-------------------------+
    | dml_inserts | Number of rows inserted |
    +-------------+-------------------------+
    ```

3. 查询 `INNODB_METRICS` 表以获取 `dml_inserts` 计数器数据。因为没有执行 DML 操作，计数器值为零或 `NULL`。`TIME_ENABLED` 和 `TIME_ELAPSED` 值指示上次启用计数器的时间以及自那时起经过的秒数。

    ```sql
    mysql>  SELECT * FROM INFORMATION_SCHEMA.INNODB_METRICS WHERE NAME="dml_inserts" \G
    *************************** 1. row ***************************
               NAME: dml_inserts
          SUBSYSTEM: dml

          MAX_COUNT: 0
          MIN_COUNT: NULL
          AVG_COUNT: 0
        COUNT_RESET: 0
    MAX_COUNT_RESET: 0
    MIN_COUNT_RESET: NULL
    AVG_COUNT_RESET: NULL
       TIME_ENABLED: 2014-12-04 14:18:28
      TIME_DISABLED: NULL
       TIME_ELAPSED: 28
         TIME_RESET: NULL
             STATUS: enabled
               TYPE: status_counter
            COMMENT: Number of rows inserted
    ```

4. 向表中插入三行数据。

    ```sql
    mysql> INSERT INTO t1 values(1);
    Query OK, 1 row affected (0.00 sec)
    
    mysql> INSERT INTO t1 values(2);
    Query OK, 1 row affected (0.00 sec)
    
    mysql> INSERT
    
     INTO t1 values(3);
    Query OK, 1 row affected (0.00 sec)
    ```

    ```sql
    mysql>  SELECT * FROM INFORMATION_SCHEMA.INNODB_METRICS WHERE NAME="dml_inserts"\G

    MIN_COUNT_RESET: NULL
    AVG_COUNT_RESET: NULL
       TIME_ENABLED: 2014-12-04 14:18:28

       TIME_ELAPSED: 65

               TYPE: status_counter

    ```

6. 重置 `dml_inserts` 计数器并再次查询 `INNODB_METRICS` 表以获取 `dml_inserts` 计数器数据。之前报告的 `%_RESET` 值，如 `COUNT_RESET` 和 `MAX_RESET`，被重置为零。`COUNT`, `MAX_COUNT`, 和 `AVG_COUNT` 等值，从启用计数器时开始累计收集数据，不受重置影响。

    ```sql
    mysql> SELECT * FROM INFORMATION_SCHEMA.INNODB_METRICS WHERE NAME="dml_inserts"\G
    ```
